# User Flow (Weekly)

## Actor
Mezei

## Primary Flow
1. Sunday intake: current inventory + goals + preferences
2. Planner generates weekly plan
3. Quick review and small edits
4. Prep block execution
5. Midweek use-up adjustment
6. End-of-week feedback captured for next run

## Decision Points
- If plan feels repetitive -> adjust dislikes/likes and rerun
- If calories too high -> lower starch/fat portions and rerun
- If protein too low -> add booster meals/snacks and rerun
- If shopping list too large -> prioritize inventory-heavy meals and rerun

## Outputs consumed weekly
- weekly-plan.md for execution
- validation.txt for confidence check
- weekly feedback template for continuous tuning
